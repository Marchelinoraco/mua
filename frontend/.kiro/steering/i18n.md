# Internationalization (i18n)

## Stack

- **i18next** + **react-i18next** for translation runtime
- **i18next-browser-languagedetector** for auto-detecting user language
- Config in `src/lib/i18n.ts` — initialized once at app entry

## Supported Languages

| Code | Language   |
|------|------------|
| `en` | English (fallback) |
| `id` | Indonesian |

Language is detected from `localStorage` (`i18next-language` key) then `navigator`, and falls back to `en`.

## Namespaces

Translations are split by domain. All files live in `src/locales/<lang>/<namespace>.json`.

| Namespace    | File              | Contains                            |
|--------------|-------------------|-------------------------------------|
| `common`     | `common.json`     | Shared UI labels (save, cancel, etc.) — **default NS** |
| `auth`       | `auth.json`       | Sign in, sign up, OTP, forgot password |
| `dashboard`  | `dashboard.json`  | Dashboard page strings              |
| `navigation` | `navigation.json` | Sidebar and nav labels              |
| `settings`   | `settings.json`   | Settings page strings               |

## Usage

### In components (hook)

```tsx
import { useTranslation } from 'react-i18next'

// Default namespace (common)
const { t } = useTranslation()
t('save') // → "Save"

// Specific namespace
const { t } = useTranslation('navigation')
t('dashboard') // → "Dashboard"

// Multiple namespaces
const { t } = useTranslation(['common', 'auth'])
t('auth:signIn') // → "Sign In"
```

### Outside components (rare)

```ts
import i18n from '@/lib/i18n'
i18n.t('common:save')
```

## Conventions

- **Always use translation keys** for any user-visible text — never hardcode English strings in JSX.
- **Add keys to both `en/` and `id/`** simultaneously. Missing keys fall back to `en` but leaving them out is a bug.
- **Use the narrowest namespace** that makes sense. Don't dump everything into `common`.
- **Key naming**: camelCase, descriptive, no nesting deeper than 2 levels.
- **Adding a new feature namespace**: create `src/locales/en/<ns>.json` and `src/locales/id/<ns>.json`, import both in `src/lib/i18n.ts`, and add to the `resources` object.

## Adding a New Namespace

1. Create `src/locales/en/<ns>.json` and `src/locales/id/<ns>.json`
2. Import them in `src/lib/i18n.ts`:
   ```ts
   import enFoo from '@/locales/en/foo.json'
   import idFoo from '@/locales/id/foo.json'
   ```
3. Add to the `resources` object:
   ```ts
   en: { ..., foo: enFoo },
   id: { ..., foo: idFoo },
   ```
4. Use `useTranslation('foo')` in the relevant feature components.
