# Project Structure

## Root

- `src/` — application source
- `public/` — static assets
- `index.html` — Vite entry HTML
- Config: `vite.config.ts`, `tsconfig*.json`, `eslint.config.js`, `.prettierrc`, `components.json` (shadcn), `knip.config.ts`

## `src/` Layout

```
src/
├── main.tsx              # App entry point
├── routeTree.gen.ts      # Auto-generated by TanStack Router (do not edit)
├── routes/               # File-based routes (thin; delegate to features)
│   ├── __root.tsx
│   ├── _authenticated/   # Protected route group
│   ├── (auth)/           # Auth route group
│   ├── (errors)/         # Error route group
├── features/             # Feature modules (the bulk of app logic)
├── components/           # Shared components
│   ├── ui/               # shadcn/ui primitives (excluded from coverage)
│   ├── layout/           # App shell / layout
│   └── data-table/       # Reusable table building blocks
├── context/              # React context providers (theme, font, direction, layout, search)
├── stores/               # Zustand stores
├── hooks/                # Shared hooks (use-* naming)
├── lib/                  # Utilities (cn, cookies, error handling)
├── config/               # Static config (e.g. fonts)
├── styles/               # Tailwind / theme CSS
├── assets/               # Logos, brand icons
└── test-utils/           # Test helpers
```

## Feature Module Convention

Each folder in `src/features/<name>/` is self-contained and follows this pattern:

```
features/<name>/
├── index.tsx             # Feature entry, rendered by the route
├── components/           # Feature-specific components
│   └── <name>-provider.tsx   # Local context/state provider when needed
└── data/                 # Schemas (Zod), mock/seed data, lookup data
    ├── schema.ts
    └── data.ts
```

- Routes stay thin: a route file imports and renders the feature's `index.tsx`.
- Keep feature-specific logic inside the feature folder; promote to `src/components`, `src/lib`, or `src/hooks` only when shared across features.

## Naming Conventions

- Files and folders: **kebab-case** (e.g. `users-action-dialog.tsx`, `use-table-url-state.ts`).
- Hooks: prefix with `use-`.
- Tests: co-located as `*.test.ts(x)` next to the file under test.
- Components: PascalCase exports; one primary component per file.

## Routing

Routes are file-based via TanStack Router. Adding/moving files in `src/routes/` regenerates `routeTree.gen.ts` automatically during dev/build. Route groups use parentheses `(group)` (no URL segment) and underscores `_layout` for pathless layout routes.
